# Task List Template

## Planning

- [ ] Confirm the approved specification.
- [ ] Confirm owner, reviewers, and target release.
- [ ] Identify affected modules, data, documentation, and CI checks.
- [ ] Identify security and privacy review requirements.

## Implementation

- [ ] Update code within the approved scope.
- [ ] Preserve existing behavior outside the approved scope.
- [ ] Validate user-controlled inputs.
- [ ] Update configuration only when required.
- [ ] Keep runtime artifacts out of version control.

## Testing

- [ ] Add or update unit tests.
- [ ] Add or update regression tests.
- [ ] Run the test suite.
- [ ] Verify coverage remains compliant.
- [ ] Document manual checks when needed.

## Documentation

- [ ] Update user documentation.
- [ ] Update architecture documentation.
- [ ] Update security or operational notes.
- [ ] Update changelog or release notes.

## Review and Release

- [ ] Run pre-commit checks.
- [ ] Confirm GitLab CI passes.
- [ ] Confirm compliance checks pass.
- [ ] Complete code review.
- [ ] Tag the release when applicable.
